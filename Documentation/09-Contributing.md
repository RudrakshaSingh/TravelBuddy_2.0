We love your input! We want to make contributing to Travel Buddy as easy and transparent as possible.

## How to Contribute

### Reporting Bugs
*   Check the Issues tab on GitHub to see if the bug has already been reported.
*   If not, open a new issue. Include:
    *   Steps to reproduce.
    *   Expected behavior vs actual behavior.
    *   Screenshots if applicable.

### Suggesting Enhancements
*   Open an Issue with the tag `enhancement`.
*   Explain your idea and why it would be useful.

### Pull Requests
1.  **Fork** the repo and create your branch from `main`.
2.  **Clone** your fork.
3.  Make sure you have the latest dependencies (`npm install`).
4.  If you've added code that should be tested, add tests.
5.  Ensure the test suite passes.
6.  **Format** your code.
7.  Commit your changes using descriptive commit messages.
8.  Push to your fork.
9.  Open a Pull Request.

## Code Style
*   We use **Prettier** for code formatting.
*   We use **ESLint** for linting.
*   Please ensure your code has no typed errors (TypeScript).
